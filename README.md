# 📱 LOTI: Local OTP Testing Infrastructure

[![NPM Version](https://img.shields.io/npm/v/@yourusername/loti-client.svg)](https://www.npmjs.com/package/@yourusername/loti-client)
[![Docker Hub](https://img.shields.io/docker/pulls/yourusername/loti-server.svg)](https://hub.docker.com/r/yourusername/loti-server)
[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)

LOTI is a local-first OTP testing infrastructure that allows developers to test actual SMS and OTP flows in their local environments **for free**, bypassing expensive APIs like Twilio during development. 

By bridging a robust Dockerized Node.js/Redis queue with your physical Android device, LOTI delivers real SMS messages to real phones directly from your `localhost`.

---

## 🚀 The 3-Minute Quickstart

You don't need to clone this repository or manually configure Docker. The LOTI CLI handles everything.

### 1. Install the CLI Globally
```bash
npm install -g @dipayan_/loti-cli
```
### 2. Ignite the Infrastructure
Navigate to an empty folder or the root of your current project and initialize the LOTI backend:

```bash
loti init
```
The CLI will automatically find an open port, generate your configuration files, and pull the pre-built backend from Docker Hub.

### 3. Connect the SDK to your App
Install the LOTI Client inside your actual Node.js application:

```bash
npm install @dipayan_/loti-client
```
## 📱 The Android Worker App (Hardware Bridge)

To actually dispatch the SMS messages from your local queue, LOTI requires our companion Android application. This app acts as a secure Socket.io worker that listens to your Dockerized backend and triggers the native Android SMS manager.

**📥 [Download the latest LOTI Android APK](https://github.com/dipa007/LOTI/releases/tag/v1.0.0t)**

> **Note on Open Source:** The LOTI CLI, Node SDK, and Dockerized Server are 100% open-source. The Android worker app source will be open-sourced in a future release. For now, a prebuilt APK is provided for easier setup.

### Connecting Your Device
1. Install the APK on your Android testing device.
2. Ensure your phone and your development machine are on the same Wi-Fi network.
3. Run `ipconfig` in your terminal and take your IPv4 address .
4. Open the mobile app and Enter your machine’s IPv4 address and the LOTI server port. (eg. http://10.51.51.103:51785 )

## 💻 Usage
LOTI is designed to be a drop-in replacement for production SMS providers during your local testing phase.

### Sending an OTP
```JavaScript
import { LotiClient } from '@dipayan_/loti-client';

// Point the SDK to the port generated by `loti init`
const loti = new LotiClient({
  endpoint: 'http://localhost:59338' // from loti-server
});

async function requestLogin() {
  const response = await loti.sendOTP({
    phone: '+919876543210',
    message: 'Your LOTI verification code is {{OTP}}. Do not share this.',
    ttl: 300,      // Expiration time in seconds (Default: 300)
    retries: 3     // BullMQ queue retries on failure (Default: 3)
  });

  console.log(`Job queued! ID: ${response.jobId}`);
}
```

### Verifying an OTP
```JavaScript
async function verifyCode(userPhone, userInput) {
  try {
    const response = await loti.verifyOTP({
      phone: userPhone,
      otp: userInput
    });
    
    if (response.verified) {
      console.log("Authentication successful!");
    }
  } catch (error) {
    console.error("Invalid or expired code.");
  }
}
```

## 📊 The Command Center Dashboard
LOTI comes with a built-in React dashboard served directly from the Docker container.

To view your real-time queue, failure logs, and connected Android device, simply run:
```bash
loti dashboard
```
(Or manually open the HOST_PORT defined in your .env file in your browser , if PORT is provided by you).

## ⚙️ Configuration
When you run loti init, a .env file is generated in your folder. You retain absolute control over the infrastructure's limits. The Docker container will instantly respect these values on boot.

```
# The external port the server and dashboard run on
HOST_PORT=59338

# Infrastructure rate limiting (max requests per TTL window)
GLOBAL_RATE_LIMIT=3

# The time (in ms) the queue waits before retrying a failed SMS
RETRY_BACKOFF_INMS=5000
```

## 🏗️ Architecture Under the Hood
LOTI is designed for resilient local OTP workflows on localhost.

<img width="1402" height="1122" alt="Image" src="https://github.com/user-attachments/assets/2be674da-1061-4dab-805c-59116e778cba" /> 

#### 1. The Client SDK: 
 A lightweight wrapper that posts requests to the local server.

#### 2. The Server & Queue:
 A Node.js backend using BullMQ and Redis to orchestrate job delivery, manage TTL expirations, and enforce strict server-side rate limits.

#### 3. The Hardware Bridge:
 A persistent Socket.io connection linking the Dockerized queue to a physical Android Worker app, executing the native SMS dispatch.

## Why LOTI ❓

During development, OTP providers like Twilio cost money.
Mock OTPs don’t simulate real SMS delivery.

LOTI solves this by:
- using your own SIM card
- running fully locally
- providing real SMS behavior
- - costs zero money beyond your existing SMS/mobile plan

Also Reliability Features like : 

- Redis-backed OTP persistence
- BullMQ retries
- delivery tracking
- failure logging
- dashboard observability
